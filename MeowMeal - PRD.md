喵膳 (MeowMeal) 产品需求文档 (PRD)  
一、 产品名称与定位  
• **产品名称**：喵膳 (MeowMeal) \- 猫咪科学自制食谱管家  
• **产品定位**：聚焦猫咪生骨肉/熟自制喂养场景，以 NRC、AAFCO 等科学营养标准为核心依据，为用户提供一站式食谱定制、食材管理、营养分析及喂养指导的工具型 APP，解决猫咪熟自制食谱制定繁琐、营养配比不科学、食材库存管理混乱等问题。  
二、 用户流程 (User Flow)

1. **初始化与建档**：用户注册/登录 \-\> 进入个人中心创建【多猫咪档案】（填写姓名、年龄、体重、绝育状态、过敏源及忌口等） 。  
2. **录入食材库存**：采购食材 \-\> 进入【冰箱库存】 \-\> 使用【OCR 智能识别】上传小票/截图 \-\> 系统提取关键信息并入库打标签。  
3. **定制食谱**：进入【食谱】模块 \-\> 选择目标猫咪（可多选）与营养标准 \-\> 选择“按需定总量”或“按库存反推”模式生成食谱 \-\> 查看营养分析与饼状图 \-\> 使用 AI 或手动替换不满意食材 \-\> 保存食谱。  
4. **执行与核销**：点击“执行食谱” \-\> 系统自动从冰箱扣除对应食材库存（若使用了冰箱食材）并记录周期。  


三、 核心功能模块详细需求与验收标准  
3.1 首页模块 (Dashboard)  
• **功能描述**：提供全局数据概览与快捷功能入口。  
◦ **执行中食谱卡片**：卡片中展示执行天数、食谱信息，点击可进入详情。  
◦ **冰箱卡片**：卡片中提供过期食材提示（跳至冰箱） 
◦ **宠物卡片**：卡片中展示当前宠物档案（支持多只），并支持快捷添加宠物。 
◦ **账单卡片**：卡片中展示当前累计话费，点击可进入详情
◦ **食材/补剂卡片**：点击展示所有食材/补剂的营养元素含量，见对应的excel
• **验收标准**：数据需与各子模块实时同步，快捷入口跳转准确无误，卡片设计、排布合理，展示要素无遗漏。  
3.2 食谱定制与管理模块（核心）  
• **功能描述**：围绕食谱的“制定-分析-编辑-执行-周期管理”全流程设计。  
◦ **多维度定制**：支持 NRC/AAFCO 标准。根据猫咪档案生成食谱（支持多只猫共用一个食谱）。提供**按需定总量**（输入每只猫咪每日喂食克数加喂食天数，系统自动计算并展示总量并分配比例（红肉、白肉、鱼、肝脏、青口贝、心脏、其他内脏的具体分配比例），制定相应食材及重量）和**按库存反推**（优先调用冰箱食材和重量，达到总克数，若缺乏某种肉则提醒用户补充）两种模式。**按库存反推**需要考虑食材重量，如购买兔腹肉1kg，最好使用500g或1kg这样的重量进行计算。 
◦ **食谱生成原则**：生成食谱的原则是避开过敏肉类，红肉占总肉量比20-30%，白肉占比鱼类占总肉量比8-10%，青口贝占总肉量比5% 肝脏占总肉量比5-10% 心脏占总肉量比5-20%  其他内脏占总肉量比5%
◦ **精细化管理与替换**：严格区分食材与补剂（自动计算 natureplus、nowsfood 等常见补剂如柠檬酸钙、碳酸钙、碘化钾等不同补剂所需克数，已提供标准数据表见MeowMeal-Common Supplement Content Table）。支持**同类型食材快速替换**（如鸡肝换鸭肝）或手动替换。 
◦ **食谱展示**： 最终的食材应列举肉类部位及重量，如红肉-猪腿肉-500g
◦ **多维度统计分析**：自动生成蛋白质/脂肪/碳水占比饼状图（展示克数及比例）、食材品类（心、肝、红肉、白肉、鱼、其他）占比的饼状图（展示克数及比例）。 
◦ **食谱达标计算**：切换标准可查看营养素“推荐含量”、“当前含量”及“是否达标”（通过红黄绿三色标记）。NRC、AAFCO两种营养标准见**MeowMeal-Nutritional element standards  NRC+AAFCO.xlsx**。当前含量需要根据食谱中的肉类和重量，自动计算营养素含量，每种肉类的营养素含量见**MeowMeal-Common Nutritional Elements in Meat.xlsx**。
◦ **补剂添加**：用户手动选择补剂品牌（品牌见MeowMeal-Common Supplement Content Table）），选择后，系统自动推荐添加含量；用户可手动修改
◦ **周期与执行**：默认 3-4 周为一周期，点击执行后自动扣除冰箱库存。  
• **验收标准**：食谱中食材分配比例依据食谱生成原则，比例在合理范围内；枚举功能均实现；营养计算结果绝对精准；两种定制模式下差额计算无误；食材替换需符合同类营养同类部位的逻辑；执行后必须触发库存精准扣减。  
3.3 冰箱库存智能管理模块  
• **功能描述**：解决食材临期浪费与库存不清问题。  
◦ **OCR 智能入库**：支持上传采购凭证（小票、截图），批量/自动提取并识别**肉的种类、部位、重量、价格、生产日期、截止日期**等信息完成录入，支持手动修正。  
◦ **智能分配与状态标记**：生成食谱时系统**优先分配即将到期的食材**。库存按“正常、临期、已过期、库存不足”进行状态标记。  
◦ **统计与查询**：支持按肉源、部位分类展示，按日期排序，及关键词精准查询。  
• **验收标准**：OCR 需具备高识别率及关键字段准确映射能力；临期预警与优先分配逻辑顺畅；扣减时出现不足需强提示。  

3.4 个人中心模块  
• **功能描述**：账号与系统基础设置。  
◦ **多猫咪档案管理**：为每只猫咪设置头像（手动上传图片）、姓名、体重、性别、绝育、过敏源、忌口、、健康需求（如护肾、减脂、低磷等）及肉类偏好等。  
◦ **账号管理**：用户头像设置、昵称修改，账号秘密修改等

四、 技术要求 (开发依赖文档)  
为了顺利实现上述功能，技术团队需要梳理并输出以下核心技术文档：

1. **营养算法与配比逻辑说明文档**：核心算法。详细定义 NRC/AAFCO 在各阶段的参数表见MeowMeal-Nutritional element standards NRC+AAFCO.xlsx，明确如何根据猫咪体重自动计算卡路里需求，进而换算为肉类比例、微量元素含量见MeowMeal-Common Nutritional Elements in Meat.xlsx及常见补剂含量表的数学模型。  
2. **OCR 接口对接与数据映射文档**：定义第三方 OCR 图像识别 API 对接规范，以及提取“品类、重量、日期、金额”等非结构化文本并清洗、映射入系统数据库字段的策略。  
3. **AI 推荐与 NLP 策略文档**：说明 AI 生成个性化喂养建议的逻辑树、同类食材快速替换（如鸡心换鸭心）的底层标签匹配规则，以及 QA 助手的 prompt 约束词库。  
4. **数据库架构与 ER 图**：设计强关联的表结构，至少包含：用户表、多宠物表、营养元素字典表、食材/补剂基础参数表、动态冰箱库存表、食谱执行周期表、账单流水表。
